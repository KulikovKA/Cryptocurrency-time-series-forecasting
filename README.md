После визуального анализа мы перешли к этапу глубокой подготовки данных. Чтобы улучшить прогностическую способность моделей, были выполнены следующие преобразования:

1.  **Объединение датафреймов:**
    Исходные данные по BTC, ETH и SOL были синхронизированы по временной шкале и объединены. 

2.  **Генерация новых признаков:**
    На основе базовых цен (OHLCV) были созданы дополнительные метрики. 

3.  **Создание лагов (Lag Features):**
    Для обучения моделей временных рядов (Time Series) критически важно учитывать исторический контекст. Были сгенерированы **лаговые признаки**  Это позволяет подавать на вход модели не только текущую цену, но и её значения `n` шагов назад, формируя "окно" истории, на основе которого строится прогноз.

##  Evaluation Metrics

Для объективного сравнения моделей была реализована **кастомная функция оценки**, которая рассчитывает комплексный набор метрик. 

### 1. Классические метрики регрессии

Эти метрики показывают, насколько сильно предсказания отклоняются от реальных значений в абсолютном выражении.

* **MAE (Mean Absolute Error):** Средняя абсолютная ошибка.
    $$MAE = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i|$$

* **RMSE (Root Mean Squared Error):** Среднеквадратичная ошибка. Сильнее штрафует за большие промахи.
    $$RMSE = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2}$$

* **MAPE (Mean Absolute Percentage Error):** Средняя процентная ошибка. Удобна для интерпретации бизнеса.
    $$MAPE = \frac{100}{n} \sum_{i=1}^{n} \left| \frac{y_i - \hat{y}_i}{y_i} \right|$$

* **WMAPE (Weighted MAPE):** Взвешенная версия MAPE, более устойчивая к выбросам и значениям, близким к нулю.
    $$WMAPE = \frac{100 \sum_{i=1}^{n} |y_i - \hat{y}_i|}{\sum_{i=1}^{n} |y_i|}$$

* **Bias (CFE):** Кумулятивная ошибка для оценки систематического смещения модели.
    * $Bias > 0$: модель систематически недооценивает цену.
    * $Bias < 0$: модель переоценивает цену.
    $$Bias = \sum_{i=1}^{n} (y_i - \hat{y}_i)$$

### 2. Продвинутые метрики (Shape & Time Warping)

Криптовалютные тренды часто имеют временные сдвиги. Классические метрики (RMSE) могут показать огромную ошибку, если прогноз сдвинут всего на 1 день, даже если форма тренда угадана верно. Для решения этой проблемы использовались:

* **Z-score нормализация:** Приведение рядов к стандартному виду перед сравнением формы.
    $$z_i = \frac{x_i - \mu}{\sigma + \varepsilon}$$

* **DTW Distance (Dynamic Time Warping):** Алгоритм динамического выравнивания времени. Позволяет найти сходство между рядами с учетом нелинейных деформаций по оси времени.
    * $DTW \approx 0$: Ряды идеально совпадают.
    * $DTW > 0.5$: Слабое совпадение.

* **DTW Производных (Trend Matching):**
    Применяется к градиентам рядов ($\nabla y, \nabla \hat{y}$). Оценивает совпадение **динамики изменений** (трендов), игнорируя абсолютные уровни цен.

* **ShapeScore (Геометрическое сходство):**
    Интегральная метрика, объединяющая обычный DTW и DTW производных.
    $$ShapeScore = \frac{1}{1 + d}, \quad \text{где } d = 0.5 \cdot d_{DTW} + 0.5 \cdot d_{Deriv}$$
    * **Диапазон:** $0 \dots 1$ (чем ближе к 1, тем точнее модель повторяет форму движения рынка).
