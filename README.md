##  Evaluation Metrics

Для объективного сравнения моделей была реализована **кастомная функция оценки**, которая рассчитывает комплексный набор метрик. 

### 1. Classic regression metrics

### MAE (Mean Absolute Error)
$$MAE = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i|$$



### RMSE (Root Mean Squared Error)
$$RMSE = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2}$$


### MAPE (Mean Absolute Percentage Error)
$$MAPE = \frac{100}{n} \sum_{i=1}^{n} \left| \frac{y_i - \hat{y}_i}{y_i} \right|$$

Средняя процентная ошибка. Позволяет сравнивать рряды разных масштабов.

### WMAPE (Weighted Mean Absolute Percentage Error)
$$WMAPE = \frac{100 \sum_{i=1}^{n} |y_i - \hat{y}_i|}{\sum_{i=1}^{n} |y_i|}$$

Взвешенная MAPE. Более стабильна, чем MAPE.

### Bias (CFE)
$$Bias = \sum_{i=1}^{n} (y_i - \hat{y}_i)$$

Кумулятивная ошибка. Показывает систематическое смещение:
- Bias ≈ 0: нет смещения 
- Bias > 0: модель недооценивает
- Bias < 0: модель переоценивает

### 2. Shape & Time Warping

### Z-score нормализация
$$z_i = \frac{x_i - \mu}{\sigma + \varepsilon}$$

Приводит ряды к стандартному виду (среднее = 0, стд. отклонение = 1).


### DTW Distance (Dynamic Time Warping)
Динамическое выравнивание временных рядов с учетом временных сдвигов:

$$D[i,j] = |a_i - b_j| + \min(D[i-1,j], D[i,j-1], D[i-1,j-1])$$

Нормализованное: $d_1 = \frac{DTW(a,b)}{n}$
**Диапазон:** 0 до ∞

**Интерпретация:**
- **DTW ≈ 0** = Ряды идеально совпадают (форма и уровни)
- **DTW = 0.1-0.3** = Очень хорошее совпадение 
- **DTW = 0.3-0.5** = Хорошее совпадение 
- **DTW > 0.5** = Слабое совпадение 

### DTW производных
$$d_2 = \frac{DTW(\nabla y, \nabla \hat{y})}{n-1}$$

**Диапазон:** 0 до ∞

**Интерпретация:**
- **DTW_deriv ≈ 0** = Одинаковая **динамика изменений** (тренды совпадают)
- **DTW_deriv = 0.1-0.2** = Очень хорошее совпадение тренда 
- **DTW_deriv = 0.2-0.4** = Хорошее совпадение тренда
- **DTW_deriv > 0.4** = Разные тренды 


Оценивает совпадение скоростей изменения (трендов).


### Shape Score (геометрическое сходство)
$$ShapeScore = \frac{1}{1 + d}, \quad d = 0.5 \cdot d_1 + 0.5 \cdot d_2$$

**Диапазон:** 0 до 1
- 1.0 = идеальное совпадение формы
- 0.5 = среднее совпадение
- ≈0 = формы совершенно разные

Не зависит от масштаба и амплитуды ряда.



## Pipeline description


Для обеспечения чистоты эксперимента и возможности сравнения результатов, ко всем трем активам (BTC, ETH, SOL) применялся единый стандартизированный цикл обработки данных и обучения.

Процесс анализа для каждой валюты состоит из 6 последовательных этапов:

### 1. Предобработка данных (Preprocessing)
* **Логарифмирование целевой переменной ($y$):** Для стабилизации дисперсии и приведения ряда к более стационарному виду использовалось преобразование $np.log(y)$. Это особенно важно для криптовалют с их экспоненциальным ростом.
* **Нормализация (Scaling):** Признаки были приведены к единому масштабу с помощью `StandardScaler` ($\mu=0, \sigma=1$), чтобы ускорить сходимость градиентного спуска.

### 2. Формирование тензоров (Sliding Windows)
Временной ряд преобразовывался в структуру для обучения с учителем (Supervised Learning) методом **скользящего окна**:
* На вход подается последовательность длиной `N` дней (Lookback period).
* На выходе прогнозируется значение на `N+1` день.

### 3-5. Обучение моделей (Model Training)
Для каждого актива последовательно обучались и тестировались три архитектуры:

####  3. BiLSTM (Bidirectional LSTM)
* **Архитектура:** Двунаправленная LSTM, которая смотрит на последовательность как от прошлого к будущему, так и обратно.
* **Цель:** Попытка уловить контекст не только из предшествующих значений, но и общей структуры окна.
* **Вывод:** Графики обучения (Loss) и предсказания на тестовой выборке (1 месяц vs Вся история).

####  4. GRU (Gated Recurrent Unit)
* **Архитектура:** Упрощенная версия рекуррентной сети без выходного гейта.
* **Цель:** Проверка гипотезы, что более простая модель лучше справится с шумом и быстрее обучится без переобучения.

####  5. LSTM (Long Short-Term Memory)
* **Архитектура:** Классическая полносвязная LSTM сеть.
* **Цель:** Базовый бенчмарк для работы с долгосрочными зависимостями во временных рядах.

### 6. Сравнительный анализ (Comparison)
Финальный этап включает:
* Сводный график предсказаний всех трех моделей на одном полотне.
* Таблицу с метриками (RMSE, MAPE, ShapeScore) для выявления лидера.

### 3. Эксперимент 1: BiLSTM (Bidirectional LSTM)

Первой мы исследовали двунаправленную рекуррентную сеть (BiLSTM). В отличие от стандартных моделей, эта архитектура обрабатывает входную последовательность одновременно в двух направлениях: от прошлого к настоящему и обратно. Это позволяет модели улавливать контекст, который может быть упущен при однонаправленном анализе.

#### 3.1 Архитектура сети
Конфигурация модели была спроектирована для борьбы с переобучением и выделения сложных паттернов:
* **Входной слой:** Принимает временные окна фиксированного размера.
* **Скрытые слои:** Два последовательных слоя **Bidirectional LSTM** (по 50 нейронов в каждом), которые извлекают признаки из временного ряда.
* **Регуляризация:** После каждого рекуррентного слоя применяется **Dropout (20%)**, что отключает часть нейронов и предотвращает запоминание шума.
* **Выход:** Полносвязный слой (Dense) с одним нейроном для прогнозирования конечной цены.

#### 3.2 Динамика обучения
Обучение проводилось с использованием оптимизатора **Adam** и функции потерь **MSE**. Для предотвращения переобучения использовался механизм ранней остановки (Early Stopping), который прерывал процесс, если ошибка на валидации переставала снижаться.

### 4. Эксперимент 2: GRU (Gated Recurrent Unit)

Вторым этапом мы протестировали сеть GRU. Это упрощенная вариация LSTM, которая не имеет выходного гейта (output gate). Гипотеза заключалась в том, что более "легкая" модель может быстрее обучаться и быть менее склонной к переобучению на шумных данных, однако ей может не хватить сложности для описания волатильности Биткоина.

#### 4.1 Архитектура сети
Архитектура зеркальна предыдущему эксперименту для честности сравнения, но использует другие рекуррентные ячейки:
* **Входной слой:** Скользящее окно данных.
* **Скрытые слои:** Два последовательных слоя **GRU** (по 50 нейронов), которые сохраняют память о предыдущих состояниях.
* **Регуляризация:** Аналогичный **Dropout (20%)** после каждого слоя.
* **Выход:** Полносвязный слой (Dense) для финальной регрессии.

#### 4.2 Динамика обучения
Использовались те же гиперпараметры: оптимизатор **Adam**, функция потерь **MSE** и механизм **Early Stopping**.

### 5. Эксперимент 3: LSTM (Long Short-Term Memory)

В качестве финального эксперимента для BTC была выбрана классическая сеть LSTM. Это "золотой стандарт" в обработке временных рядов, разработанный специально для решения проблемы затухающего градиента и запоминания долгосрочных зависимостей.

#### 5.1 Архитектура сети

* **Входной слой:** Временное окно.

* **Регуляризация:** **Dropout (20%)** для генерализации.
* **Выход:** Слой Dense(1).

### 3. Эксперимент 1: BiLSTM (Bidirectional LSTM)

Первой для Эфириума мы исследовали двунаправленную рекуррентную сеть (BiLSTM). Данная архитектура позволяет учитывать контекст не только предыдущих дней, но и общую структуру окна за счет прохода в обратном направлении. Для более структурированного движения ETH это могло дать преимущество.

#### 3.1 Архитектура сети
Конфигурация модели была спроектирована для борьбы с переобучением и выделения сложных паттернов:
* **Входной слой:** Принимает временные окна фиксированного размера.
* **Скрытые слои:** Два последовательных слоя **Bidirectional LSTM** (по 50 нейронов в каждом), которые извлекают признаки из временного ряда.
* **Регуляризация:** После каждого рекуррентного слоя применяется **Dropout (20%)**, что отключает часть нейронов и предотвращает запоминание шума.

Обучение проводилось с использованием оптимизатора **Adam** и функции потерь **MSE**. Для предотвращения переобучения использовался механизм ранней остановки (Early Stopping), который прерывал процесс, если ошибка на валидации переставала снижаться.
